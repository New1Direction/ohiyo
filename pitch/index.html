Under the hood

No hand-rolled crypto. The real thing.

HandshakeX3DH

Asynchronous key agreement — message someone who's offline, forward-secret from the first byte.

1:1 sessionsDouble Ratchet

Per-message keys with forward secrecy & post-compromise recovery, via the audited libsignal.

GroupsSender keys + epochs

Every membership change bumps an epoch and rekeys; a removed member can't read a single future message.

TrustSafety numbers

Out-of-band verification across all of a contact's devices; a swapped key raises a loud banner.

At restVault + backup

Desktop seals keys in an AES-256-GCM vault; recovery is a self-custody code (PBKDF2 → AES-GCM).

The server only ever relays ciphertext. It moves sealed envelopes between devices, enforces membership, and stores nothing it can read — verified by a Rust integration suite that proves a removed member loses access and a pinned identity can't be silently swapped.
